# Лабораторна робота №3: Реалізація власного blockchain (DC_Lab_3)

## Огляд та призначення продукту
Цей проєкт є спрощеною реалізацією платіжної системи на базі технології блокчейн. Це навчальна модель, створена для вивчення фундаментальних принципів розподілених реєстрів.

**Цілі проєкту:**
* Розуміння механізмів зв’язування блоків через криптографічні хеш-функції.
* Інтеграція власної реалізації алгоритму хешування (SHA-1), розробленої під час Лабораторної роботи №1.
* Забезпечення цілісності даних: зміна одного блоку робить недійсною всю наступну чергу.
* Вивчення базової логіки транзакцій та управління балансами користувачів.

## Архітектура системи (Опис класів)
Програмна логіка розділена на незалежні класи, що забезпечує чистоту архітектури та єдину відповідальність кожного компонента:

* **`Hash`**: Відповідає за криптографічне хешування. Містить власну математичну реалізацію алгоритму SHA-1 (з Лабораторної №1), яка перетворює будь-які вхідні дані на унікальний 160-бітний хеш.
* **`Account`**: Представляє гаманець (користувача) в системі. Зберігає ім'я користувача, унікальний ідентифікатор (на базі GUID) та поточний локальний баланс.
* **`Operation`**: Описує атомарну дію переказу монет. Містить посилання на відправника (`Sender`), отримувача (`Receiver`), суму (`Amount`) та метод `Verify()` для базової перевірки (чи не є сума від'ємною та чи вистачає коштів).
* **`Transaction`**: Контейнер, що групує одну або декілька операцій. Використовує унікальний параметр `Nonce` для захисту від дублювання однакових переказів та генерує власний `TransactionId` (хеш).
* **`Block`**: Зв'язна ланка ланцюга. Об'єднує список підтверджених транзакцій та **обов'язково** зберігає хеш попереднього блоку (`PrevHash`). На основі цих даних формує власний унікальний `BlockId`.
* **`Blockchain`**: Головний клас-контролер. Керує історією блоків (`BlockHistory`), глобальним станом балансів мережі (`CoinDatabase`), реєстром опрацьованих транзакцій (`TxDatabase`) та містить жорстку логіку валідації нових блоків перед їх додаванням.

## Взаємодія
Продукт реалізований як консольний додаток мовою **C#**. 
Весь ввід та вивід здійснюється через термінал. Точкою входу є клас `Program`, який ініціалізує `Blockchain`, створює користувачів та запускає симуляцію переказів.

## Функції продукту
1. **Управління акаунтами:** Створення користувачів з унікальними ідентифікаторами та зрозумілими іменами.
2. **Faucet (Кран):** Механізм нарахування початкових тестових монет користувачам через системний виклик.
3. **Проведення платежів:** Переказ TC (TestCoin) між акаунтами з автоматичною перевіркою достатності коштів.
4. **Кастомне хешування:** Замість використання стандартних бібліотек C#, для генерації ID блоків та транзакцій використовується власна реалізація SHA-1.
5. **Формування ланцюжка:** Групування транзакцій у блоки із жорстким криптографічним зв'язком (`PrevHash`).
6. **Аудит системи:** Метод `PrintBlockchain()` візуалізує весь ланцюжок блоків у консолі для перевірки зв'язності та цілісності.

## Вимоги до безпеки
* **Зв'язність блоків:** Кожен блок жорстко прив'язаний до хешу попереднього. Будь-яка спроба змінити дані в минулому змінить хеш і зруйнує ланцюжок.
* **Валідація транзакцій (Двоетапна):** Спочатку блокчейн перевіряє всі операції в новому блоці на валідність (достатність балансу, відсутність дублікатів). Лише у разі 100% успіху застосовуються зміни до глобальної бази `CoinDatabase`.
* **Захист від дублювання (Double-spending):** Використання унікального поля `nonce` у транзакціях та перевірка через `TxDatabase` запобігає повторному використанню однієї й тієї ж транзакції.

## Обмеження
* **Локальність:** Система працює виключно в оперативній пам'яті (без децентралізації, P2P-мережі та зовнішніх баз даних).
* **Відсутність майнінгу:** Консенсус досягається шляхом прямої локальної валідації при додаванні блоку.
* **Криптографічні підписи:** З метою спрощення у цій версії не використовуються (ідентифікація відправника та отримувача відбувається за їхніми об'єктами).
* **Зберігання даних:** Стан блокчейну скидається відразу після завершення роботи консольної програми.
